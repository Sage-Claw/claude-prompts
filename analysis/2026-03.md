### New Capabilities/Instructions Added

1. **Tool-preference rules (v2.1.86)**: Explicit instructions to use `Read` instead of `cat/head/tail/sed`, `Edit` instead of `sed/awk`, `Write` instead of `echo/heredoc`, `Glob` instead of `find`, `Grep` instead of `grep/rg`. Bash is now reserved for "system commands only." This is a significant philosophical shift — Claude Code is being trained away from shell-scripting habits.

2. **Scratchpad directory (v2.1.86)**: Session-specific temp directory instead of `/tmp`. Labeled as "IMPORTANT" in the prompt. Signals Anthropic's move toward better session isolation and cleanup.

3. **`! <command>` shorthand (v2.1.86)**: Users can type `! gcloud auth login` to run interactive commands directly. Claude is instructed to suggest this pattern when the user needs to run something themselves.

4. **Third-party upload warning (v2.1.78)**: New safety instruction — uploading to diagram renderers, pastebins, gists may be cached or indexed. Shows growing awareness of data leakage vectors.

5. **GitHub link format (v2.1.86)**: Instructed to use `owner/repo#123` format for issue/PR references so they render as clickable links.

2. **Ask-on-denial instruction (added v2.1.72, removed v2.1.75)**: A 4-day experiment where Claude was told to use the ask tool to clarify when a tool call was denied. Quickly reverted — possibly caused too many unnecessary clarification requests.

### Philosophy Shifts

**From shell tool to structured agent.** The biggest story of March is the tool-preference rules. Anthropic is clearly pushing Claude Code to use its native tools (Read, Edit, Write, Glob, Grep) instead of falling back to bash commands. This reduces the blast radius of operations, makes actions more reviewable by users, and positions Claude Code as a proper agent rather than a sophisticated shell script generator.

**Debugging over brute-force.** The refined instruction in v2.1.86 — "If an approach fails, diagnose why before switching tactics — read the error, check your assumptions, try a focused fix" — replaces a simpler "don't brute-force" rule. The nuance added: "don't abandon a viable approach after a single failure either." This acknowledges that the previous instruction may have caused Claude to give up too quickly.

**Output efficiency convergence.** After a week of churn (Mar 3–4), the output guidance settled into a stable form: focus on decisions, milestones, blockers. Short sentences over explanations. The CRITICAL-level "go straight to the point" survived.

**Growing data-sensitivity awareness.** The third-party upload warning is new territory — the prompt now explicitly warns about pastebins and diagram renderers being potential data leaks. This aligns with enterprise security concerns.

### Interesting Patterns

- **Rapid A/B testing visible in the changelog**: The ask-on-denial feature lasted exactly 4 days. The output efficiency format changed 3 times in 2 days. Anthropic is clearly iterating fast and reverting quickly when things don't work.
- **The "complexity balance" wording got refined twice**: From "no speculative abstractions" (original) to adding "but no half-finished implementations either" — a clear response to users reporting Claude under-delivering when told to keep things simple.
- **Prompt injection defense is now permanent**: The March restructure (building on the Feb 24 addition) made prompt-injection flagging a standard part of the system prompt.

## Cross-Tool Trends

### 1. Output Efficiency Is Universal
Claude Code spent a week iterating on output brevity rules. This mirrors a broader industry pattern — all tools are converging on "less narration, more action." Users of agentic tools want results, not explanations of what the agent is about to do.

### 2. Tool Abstraction Over Shell
Claude Code's new tool-preference rules (use Read not cat, Edit not sed) represent a trend toward **purpose-built tool abstractions** over raw shell access. This makes agent actions more auditable, reversible, and controllable — a pattern Cline has used from the start with its structured tool approach.

### 3. Safety Layering Deepens
March saw Claude Code add third-party upload warnings and refine its "execute with care" section. The prompt-injection defense from late February is now settled. Security is becoming a first-class prompt concern, not an afterthought.

### 4. Session Isolation
The scratchpad directory in Claude Code is a clear signal: agents need their own isolated workspace. This prevents temp file collisions between sessions and reduces the chance of agents stepping on each other's work in multi-agent setups.

### 5. Rapid Experimentation & Reversion
The ask-on-denial 4-day experiment in Claude Code shows a pattern: ship it, measure it, revert if wrong. This is healthy prompt engineering — treating system prompts as living documents that can be A/B tested.
